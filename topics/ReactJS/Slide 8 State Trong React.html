Hiểu về State trong ReactJS

Dữ liệu nội bộ, sự khác biệt với Props và cách quản lý

State là gì?

Dữ liệu nội bộ thay đổi theo thời gian

  • `State` là một đối tượng JavaScript giữ dữ liệu riêng tư của một component.
  • Dữ liệu này có thể thay đổi trong suốt vòng đời của component.
  • Khi `State` thay đổi, React sẽ tự động re-render component đó và các component con liên quan.

State và Props

Sự khác biệt cốt lõi

  • **Props:**
    • Truyền từ component cha xuống con.
    • Không thể thay đổi bởi component con (immutable).
    • Dùng để cấu hình và truyền dữ liệu.
  • **State:**
    • Quản lý bên trong component.
    • Có thể thay đổi bởi chính component đó (mutable).
    • Dùng để quản lý dữ liệu động.

Quản lý State

Khởi tạo và cập nhật

  • **Khởi tạo (Function Components):** Sử dụng `useState` Hook.
    • `const [value, setValue] = useState(initialValue);`
  • **Cập nhật (Function Components):** Sử dụng hàm setter từ `useState`.
    • `setValue(newValue);`
    • Luôn cập nhật `State` một cách bất biến (immutable).
    • Cập nhật `State` là asynchronous.
