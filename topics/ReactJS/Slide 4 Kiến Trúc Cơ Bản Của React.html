React Core Concepts: Components & Virtual DOM

Nền tảng xây dựng giao diện người dùng hiệu quả và tối ưu hiệu suất.

Mô Hình Component-based

Xây dựng UI bằng các khối độc lập

  • Giao diện được chia thành các Component nhỏ, độc lập.
  • Mỗi component quản lý trạng thái riêng và có thể tái sử dụng.
  • Tăng tính modularity, dễ bảo trì và mở rộng ứng dụng.
  • Ví dụ: nút, thanh điều hướng, form nhập liệu.

Virtual DOM là gì?

Bản sao nhẹ của DOM thực

  • Một biểu diễn ảo của Document Object Model (DOM) trong bộ nhớ.
  • React tạo VDOM từ các component của bạn.
  • Không trực tiếp thao tác với DOM thực, giúp tăng tốc độ.
  • Là cơ chế trung gian để React tối ưu cập nhật UI.

Cách Virtual DOM Hoạt Động

Tối ưu hiệu suất qua cơ chế Reconciliation

  • Khi trạng thái thay đổi, React tạo VDOM mới.
  • So sánh VDOM mới với VDOM cũ để tìm ra các thay đổi ("diffing").
  • Chỉ cập nhật những phần tử thay đổi thực sự lên Real DOM.
  • Tránh việc re-render toàn bộ trang, mang lại hiệu suất cao.
